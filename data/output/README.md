# output/

Store final data products used by reports, notebooks, or downstream apps.

Note: this `data/output/` folder is for datasets.
It is different from the repository-level `outputs/` folder, which is treated as legacy/optional for generated materials (for example, figures).

Rules:
- Include only publish-ready or consumption-ready datasets.
- Use purpose-based names, for example `output_crop_risk_scores_20260320.csv`.
- Keep output reproducible from scripts and notebooks.
